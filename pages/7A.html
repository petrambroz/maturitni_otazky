Členské a statické proměnné a metody

  • Statická proměnná – existuje nezávisle a objektech
    • Patří třídě ve které byla vytvořena
  • Statická metoda – lze ji volat bez nutnosti vytvořit objekt
    • Má přístup pouze ke statickým proměnným třídy
  • Členská proměnná – vztahuje se ke konkrétnímu objektu
  • Členská metoda – lze ji volat pouze z konkrétního objektu

Modifikátory přístupu

  • Public – umožňuje přístup k prvku odkudkoli -> lze ho i měnit odkudkoli
  • Protected – umožňuje přístup pouze z třídy, kde byl deklarován a z jejích potomků
  • Private – přístup pouze z dané třídy

Zapouzdření

  • Aby nebylo možné u některých prvků je kdykoliv měnit, ale potřebujeme k nim přístup i z jiných míst než z dané třídy (private/protected by nám přístup neumožnilo), využijeme takzvané gettery a settery

private int vek;
public int getVek() { //getter
	return this.vek;}
public void setVek(int vek) { //setter
	this.vek = vek;}

Dědičnost

  • Každé třídě můžeme vytvořit podtřídy, které budou mít stejné vlastnosti, proměnné a konstruktory jako rodičovská třída, plus navíc své vlastní
  • Využití: můžeme využít starší kód, který již máme vyzkoušený a odladěný k vytvoření nové třídy, u které potřebujeme nějaké funkce navíc

this, super

  • this – potřebujeme pojmenovat stejně předaný argument a proměnnou
    • Aby bylo jasné, že to, co obsahuje this se vztahuje ke konkrétní instanci
public Pes(String barva, int vek) {
	this.barva = barva;
	this.vek = vek;}
  • Super – pokud chceme využít konstruktor rodičovské třídy, zavoláme ho pomocísuper(argument)

public Robot(String barva, int kapacitabaterie) {
  super(barva);
  this.kapacitabaterie = kapacitabaterie;
}
